==================================================================================================================
 Проверьте созданный бэкап:
bash
# Смотрим список файлов
ls -la backup_*.tar.gz

# Проверяем содержимое архива
tar -tzf backup_$(date +%Y%m%d).tar.gz | head -20

# Узнаём размер
du -sh backup_*.tar.gz
📁 Что сохранилось в бэкапе:
/app/user_logs/ - все логи переписки с пользователями

/app/exports/ - CSV файлы с экспортом пользователей

/app/users.db - база данных пользователей

🔄 Как восстановиться из бэкапа:
bash
# Останавливаем бота
sudo docker-compose down

# Восстанавливаем файлы
sudo tar -xzf backup_YYYYMMDD.tar.gz -C .

# Запускаем бота
sudo docker-compose up -d
📋 Дополнительные полезные команды:

# Создать бэкап с датой в имени

bash

mkdir -p ~/backups
